# 焊接向量计算 - 5类点详细对比

## 📊 汇总对比表

| 特性 | 普通点 | 通孔边点 | 断点 | 非基板点 | 备用面 |
|------|--------|---------|------|---------|--------|
| **英文名** | Regular Endpoint | Through-Hole Arc | Breakpoint | Non-Base Point | Fallback Face |
| **占比** | ~60-70% | ~20-30% | ~10-15% | ~5-10% | ~2-5% |
| **出现位置** | 焊接线中间 | 通孔周围 | 焊接序列端点 | 加强筋/补强板 | 选择不足时 |
| **关键特征** | 普通接触面 | 有弧形边界 | T型/多方向 | 多实体接触 | 降级备选 |
| **法线数量** | 3个 | 3个 | 2-3个 | 3个 | 1个(补充) |
| **选择优先级** | base>side>other | 特殊逻辑 | current_weld优先 | 多样化优先 | 最后降级 |
| **计算复杂度** | 中 | 高 | 中高 | 高 | 低 |
| **调用函数** | `select_regular_normals()` | `select_hole_*()` | `select_contact_face_...()` | `select_non_hole_...()` | `select_lower_level_...()` |

## 🟢 1. 普通点 (Regular Endpoint)

### 📍 定义
- 位于标准焊接路径的点
- 不在特殊位置(通孔、断点、非基板)
- 属于接触面上的点

### 🎯 物理意义
```
普通的焊接接头处
↓
通常是基板 + 两个加强筋的三面接触
↓
向量应该指向"最稳定的焊接接近方向"
```

### 🔧 计算策略

```python
def select_regular_normals(normals, up_axis):
    """
    选择: 1个base面 + 2个side面
    """
    
    # 第一步: 分类法线
    base_candidates = []
    side_candidates = []
    
    for normal in normals:
        dot_up = np.dot(normal, up_axis)
        
        # Base面: 几乎垂直向上
        if dot_up > 0.65:
            base_candidates.append((dot_up, normal))
        
        # Side面: 基本水平
        elif abs(dot_up) < 0.45:
            side_candidates.append((abs(dot_up), normal))
    
    # 第二步: 优先选择base面 (点积最大的)
    selected = []
    if base_candidates:
        best_base = max(base_candidates)[1]
        selected.append(best_base)
    
    # 第三步: 选择2个side面 (最水平的)
    side_candidates.sort()
    for _, side in side_candidates[:2]:
        if not is_duplicate(side, selected):
            selected.append(side)
    
    return selected[:3]
```

## 🔴 2. 通孔边点 (Through-Hole Arc Point)

### 📍 定义
- 点恰好在通孔的圆周边界上
- 需要特殊的垂直性检查
- 分两个子情况

### 🎯 物理意义

#### **情况1: 非基板点** (在加强筋上的通孔)
```
加强筋
    ↓
  [□] ← 点在这里 (通孔边界)
  /│\
 / │ \  ← 需要3个法向量来定向
```

#### **情况2: 基板点** (在基板上的通孔)
```
基板
  ↓
[○] ← 通孔
 ││ ← 需要确保垂直于孔
```

### 🔧 计算策略

```python
def handle_hole_arc_point(solids, solid_infos, xyz, solid_ids, hole_arc_info):
    """
    通孔特殊处理
    """
    arc_id, arc_center = hole_arc_info
    base_id = get_base_plate_id(solid_infos)
    is_non_base = base_id not in solid_ids
    
    if is_non_base:
        # 非基板: 选择长/短焊接方向
        selected = select_hole_non_base_plane_normals(
            solids, xyz, solid_ids,
            weld_edge_lengths,  # 焊接边长度
            up_axis
        )
        # 结构: [up_vector] + [long_weld_dir] + [short_weld_dir]
    else:
        # 基板: 选择接触肋面
        selected = select_hole_base_weld_direction_normals(
            solids, xyz, solid_ids,
            incident_edges,
            up_axis
        )
        # 结构: [up_vector] + [rib1] + [rib2]
    
    return selected
```

## 🟡 3. 断点 (Breakpoint)

### 📍 定义
- 焊接序列的分界点
- 标志着焊接方向/焊缝的改变
- 常见于T型接头、多路径焊接

### 🎯 物理意义
```
焊缝A ─────┐
           ├─ 断点
焊缝B ─────┘

在断点处需要特殊的向量以支持焊接路径转变
```

### 🔧 计算策略

```python
def handle_breakpoint(valid_breakpoints, edge, contact_edges):
    """
    断点处理的两种情况
    """
    
    # 情况1: 有效的T型断点 (来自当前焊缝)
    if is_valid_t_type_breakpoint(point_id):
        # 使用当前焊缝的两个接触面
        bp_solid_ids = get_current_weld_solids(edge)
        selected = select_contact_face_normals_for_solids(
            solids, xyz, bp_solid_ids,
            source="breakpoint_current_weld_contact_face"
        )
        method = "breakpoint_current_weld_two_contact_faces"
    
    # 情况2: 来自入射边的断点
    else:
        # 使用相邻焊接边的接触面
        current_edge = incident_edges[0]
        bp_solid_ids = current_edge["solid_ids"]
        selected = select_contact_face_normals_for_solids(
            solids, xyz, bp_solid_ids,
            source="breakpoint_incident_edge_contact_face"
        )
        method = "breakpoint_incident_edge_two_contact_faces"
    
    return selected
```

## 🟠 4. 非基板点 (Non-Base Point)

### 📍 定义
- 点不在基础板上，而在加强筋/补强板上
- 属于多实体接触的复杂场景
- 需要严格的遮挡检查

### 🎯 物理意义
```
加强筋
  │
  ├─ 基板
  │
  └─ 加强筋

在这种多实体交集处:
需要选择3个"不被其他实体遮挡"的法向量
```

### 🔧 计算策略

```python
def select_non_hole_non_base_three_normals(solids, xyz, incident_solid_ids):
    """
    非基板点的多样化选择
    """
    
    # 第一步: 从incident_solid_ids中提取候选法向量
    candidates = []
    for solid_id in incident_solid_ids:
        normals = face_normals_at_point(solids[solid_id], xyz, tol)
        
        # 检查是否被其他实体遮挡
        for n in normals:
            test_pt = xyz + n * 0.1
            if is_point_inside_other_solids(test_pt, solids, solid_id):
                continue  # 被遮挡，跳过
            candidates.append(n)
    
    # 第二步: 选择3个"最不相似"的向量 (多样化)
    # - 目标: 向量指向不同方向
    # - 这样合并后能得到最稳定的结果
    selected = select_max_diverse_three(candidates)
    
    return selected
```

## 🟡 5. 备用面 (Fallback Face)

### 📍 定义
- 当前选择的法向量不足3个时触发
- 使用较低优先级的面作为补充
- 是一种"应急方案"

### 🎯 物理意义
```
理想情况: 3个优先级高的法向量
         ↓
        需要降级到备用面来凑足3个
```

### 🔧 计算策略

```python
def select_lower_level_fallback_face(solids, solid_infos, xyz):
    """
    备用面的选择顺序
    """
    
    # 降级优先级
    priority_order = [
        "boundary_face",      # 一级备选
        "auxiliary_face",     # 二级备选
        "other_face",         # 三级备选
    ]
    
    for fallback_role in priority_order:
        # 尝试从该优先级获取面
        face = find_face_by_role(solids, xyz, fallback_role)
        if face and is_not_duplicate(face, selected):
            return {
                "normal": face.normal,
                "fallback_role": fallback_role,
                "fallback_source_role": face.solid_role,
                "face_index": face.index,
                "distance": distance_to_face
            }
    
    return None  # 无法找到备用面
```

## 🔀 法向量的合并 (combine_normals)

### 最后一步: 将选中的3个法向量合并为1个

```python
def combine_normals(selected):
    """
    将3个法向量合并为1个pose_normal
    """
    # 简单的向量加权平均 + 归一化
    
    # 累加所有法向量
    sum_vector = np.zeros(3)
    for normal in selected:
        sum_vector += np.array(normal)
    
    # 归一化为单位向量
    pose_normal = sum_vector / np.linalg.norm(sum_vector)
    
    return pose_normal
```

### 例子:
```
selected = [
    [0, 0, 1],        # base面 (向上)
    [1, 0, 0],        # side1 (向右)
    [0, 1, 0]         # side2 (向前)
]

sum = [0+1+0, 0+0+1, 1+0+0] = [1, 1, 1]
norm = sqrt(1²+1²+1²) = sqrt(3) ≈ 1.732

pose_normal = [1/1.732, 1/1.732, 1/1.732]
            ≈ [0.577, 0.577, 0.577]
```

## 🎓 总结

| 维度 | 说明 |
|------|------|
| **分类方式** | 根据点的位置和上下文场景 |
| **共几类** | 5类 (包括备用面) |
| **核心思想** | 不同场景用不同策略，都指向"最佳焊接接近方向" |
| **法向量数量** | 大多数选择3个，合并后得到1个pose_normal |
| **优先级系统** | base面 > side面 > other面 > fallback面 |
| **计算复杂度** | 普通点简单，特殊点复杂(需要多次check和降级) |
| **输出格式** | 单位向量 [nx, ny, nz] |
